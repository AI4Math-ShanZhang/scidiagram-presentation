Evaluation Challenge

Metric Limitations in Current Benchmarks

Why existing metrics fail to capture structural understanding

Problems with Current Metrics

Global Metrics (FID, CLIP)

Measure visual similarity, miss structural correctness

Token-Level Metrics

Order-sensitive: equivalent code with different ordering penalized

Focus on Syntax

Evaluate code correctness, not perceptual understanding

The Evaluation Gap

High FID/CLIP Scores
Visual similarity looks good

Structural Correctness
Logical & topological accuracy

Solution: We need primitive-level metrics that are order-invariant and structurally meaningful

Our Solution

Primitive-Level Evaluation Metrics

Four semantic dimensions extracted directly from code syntax

Type

Element types with domain meaning

"resistor", "benzene", "scatter"

Text

Labels, titles, annotations

with semantic positions

Color

Draw/fill colors

component-specific

Layout

Spatial arrangement

topological connectivity

Why Primitive-Level Metrics?

Order-invariant: Equivalent code with different ordering is treated equally

Semantically meaningful: Measures actual structural understanding

Edit-sensitive: Detects instruction-conditioned modifications

Extraction Method

Primitives extracted from TikZ/LaTeX syntax

Domain-specific parsers for each package

Direct comparison without rendering

Result: Multi-granularity evaluation at token-level, primitive-level, and global-level
